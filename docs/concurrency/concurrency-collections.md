# Concurrency collections

**ConcurrentHashMap** — потокобезопасная коллекция типа HashMap, реализующая интерфейс ConcurrentMap.Это достигается за счет разделения карты на сегменты, каждый из которых может быть заблокирован независимо.

**CopyOnWriteArrayList** — потокобезопасная коллекция типа ArrayList с алгоритмом CopyOnWrite. Позволяет нескольким потокам одновременно читать список, но только один поток может одновременно модифицировать список. Это достигается за счет создания новой копии списка при каждой его модификации, что может быть дорогостоящим с точки зрения использования памяти.

**CopyOnWriteArraySet** — потокобезопасная реализация интерфейса Set, использующая за основу CopyOnWriteArrayList.

**ConcurrentNavigableMap** —  расширяет интерфейс NavigableMap.

**ConcurrentSkipListMap** — потокобезопасный аналог коллекции TreeMap с сортировкой данных по ключу и с поддержкой многопоточности.

**ConcurrentSkipListSet** — реализация интерфейса Set, выполненная на основе класса ConcurrentSkipListMap.

**ConcurrentLinkedQueue** — потокобезопасная реализация интерфейса Queue. Позволяет нескольким потокам одновременно добавлять и удалять элементы из очереди без необходимости внешней синхронизации. Это достигается за счет использования алгоритма без блокировок, основанного на операциях сравнения и замены.

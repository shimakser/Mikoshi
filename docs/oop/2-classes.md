# Виды классов

**Объект** — отдельный представитель класса, имеющий конкретное состояние и поведение, полностью определяемое классом.

**Класс** — способ описания сущности, определяющий состояние и поведение, зависящее от этого состояния, а также правила для взаимодействия с данной сущностью.

---

**Абстрактный класс** — приблизительный «шаблон» по которому будут создаваться новые классы наследуясь от абстрактного.

---

**Интерфейс** — контракт — набор методов класса, доступных для использования.

Содержит:
- абстрактные методы (без реализации),
- константы (public static final).

Также может содержать методы с имлементацией:
1.  default
    `default double perimeter() {
        return 0;
    }`
2. static (не переопределяется)
   `static double square(double x) {
       return x * x;
   }`

---

**Вложенный класс** — должны создаваться только для того, чтобы обслуживать обрамляющие его классы. Если вложенный класс оказывается полезен в каком-либо ином контексте, он должен стать классом верхнего уровня.
- **Статический вложенный класс** — если ссылка на обрамляющий класс не требуется; чтобы укомплектовать связанные классы рядышком, чтобы с логической структурой было работать проще.
- **Простой внутренний класс** — существуют только у объектов, не может быть статических переменных. Если вам нужны какие-то константы или что-либо еще статическое, выносить их нужно во внешний класс. Это связано с тесной связью нестатического вложенного класса с внешним классом. У класса полный доступ ко всем приватным полям внешнего класса. Данная особенность работает в две стороны.
- **Локальный класс** — если класс необходим только внутри какого-то метода и требуется создавать экземпляры этого класса только в этом методе. Могут обращаться к локальным переменным и параметрам метода, если они объявлены с модификатором final. Имеют доступ к полям и методам обрамляющего класса. Не могут быть объявлены как private/public/protected или static.
- **Анонимный класс** — вложенный локальный класс без имени. Использование разрешено только в одном месте программы - месте его создания. Реализует лишь методы своего интерфейса или суперкласса.

---

**Records** — компактный способ создания класса.
Являются неизменяемыми по умолчанию. Все поля являются final и могут быть установлены только 1 раз через конструктор.
Неявно являются final и не могут быть расширены.

---

**Sealed class** — позволяют определять, какие классы могут наследовать от данного класса или интерфейса.
Подклассы должны быть final, sealed или non-sealed.

`public sealed class Shape permits Circle {}`

`final class Circle extends Shape {}`

Каждый класс, указанный в permits, обязан явно указать одну из следующих модификаций:
- final — больше никто не может наследоваться от него.
- sealed — он сам тоже ограничивает своих наследников.
- non-sealed — снимает ограничение, и от него можно снова свободно наследоваться

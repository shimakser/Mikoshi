# Объектно-ориентированное программирование (ООП)

Методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых
является экземпляром определенного класса, а классы образуют иерархию наследования.

### Принципы ООП
- **Inheritance** — Создание новой сущности на базе уже существующей.
- **Abstraction** — Способ выделить набор общих характеристик объекта, исключая из рассмотрения частные и незначимые.
- **Encapsulation** — Свойство системы, позволяющее объединить данные и методы, работающие с ними, в классе и скрыть детали реализации от пользователя, открыв только то, что необходимо при последующем использовании.
- **Polymorphism** — Свойство системы использовать объекты с одинаковым интерфейсом без информации о типе и внутренней структуре объекта.

### Плюсы

- ориентирована на человеческое восприятие мира.
- возможность создавать расширяемые системы.
- позволяет привнести свойство модульности.
- повторное использование кода.

### Минусы

- сложная иерархия.
- излишняя универсальность.

## Aggregation vs Composition

### Aggregation

Объекты связаны взаимоотношением «part-of» (часть).

### Composition

Более строгий вариант агрегации; дополнительно к «part-of» накладывается условие, что экземпляр может входить только в
одно целое (или никуда не входить).

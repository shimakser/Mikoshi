# NoSQL, BASE

## BASE

BASE (для распределённых систем, NoSQL):
- Basically Available — система всегда отвечает, даже если с ошибкой.
- Soft state — состояние системы может меняться без ввода (в силу eventual consistency).
- Eventual consistency — данные станут согласованными "в итоге", не обязательно сразу.

Используется в распределённых, масштабируемых системах (NoSQL, Cassandra, DynamoDB).

---

## NoSQL

NoSQL — это семейство баз данных, которые:
- не используют классическую табличную модель,
- могут не иметь жёсткой схемы,
- ориентированы на горизонтальное масштабирование,
- оптимизированы под специфические use-case'ы (например, хранение JSON-документов, графов, кешей и т.д.).
- Горизонтальное масштабирование.

NoSQL —  про гибкость, масштаб и структуру данных, неподходящую для реляционных БД.

### Основные типы NoSQL баз данных
1. Документо-ориентированные
   - Гибкие данные, подобные JSON: профили пользователей, заказы.
   - MongoDB, Couchbase.
2. Ключ-значение
   - Очень быстрый доступ по ключу: сессии, кэш.
   - Redis, DynamoDB.
3. Колонковые
   - Аналитика, big data, time series.
   - Cassandra, HBase.
4. Графовые
   - Сети, социальные графы, рекомендации.
   - Neo4j, ArangoDB.

### Почему и когда используют NoSQL
- Структура данных часто меняется (например, JSON от разных клиентов).
- Нужна масштабируемость по горизонтали.
- Большие объёмы — миллионы или миллиарды документов.
- Высокая скорость записи / отдачи (например, IoT, логи).
- Неформализованные данные (например, динамическая анкета).

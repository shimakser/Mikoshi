# Exceptions

### Исключения делятся на 2 группы

1. Ошибки, возникающие при выполнении программы в результате сбоя работы JVM, переполнения памяти или сбоя системы.
   Исключения, унаследованные из класса Error. Такой вид исключений в Java относится к unchecked на стадии компиляции. К
   этой группе также относят RuntimeException – исключения, наследники класса Exception, генерируемые JVM во время
   выполнения программы. Часто причиной возникновения их являются ошибки программирования. Написание кода по их
   обработке не является обязательным.
2. Checked исключения, предвидимые еще на стадии написания программы, для которых должен быть написан код обработки.

### try with resources

Нужен для закрытия ресурсов после работы, можно поместить все, что наследуется от Closable.
try/finally без catch: создать можно и даже отработает если ошибка непроверяемая, если проверяемая попросит или добавить
catch или обернуть в try/catch или добавить в сигнатуру метода, если не обработали выдаст ошибку.

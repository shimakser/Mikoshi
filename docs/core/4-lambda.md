# Lambda

Это набор инструкций, которые можно выделить в отдельную переменную и затем многократно вызвать в различных местах программы.

---

## Переменные

1. объявленные на уровне класса, можем получить и даже изменить. После выполнения выражения изменяется значение
   переменной.
2. переменные уровня метода можем использовать в лямбдах, но изменять их значение нельзя. Если мы попробуем это сделать,
   то среда разработки может нам высветить ошибку.

---

## Отличие от анонимного класса

| Критерий           | Анонимный класс               | Лямбда                              |
|--------------------|-------------------------------|-------------------------------------|
| Синтаксис          | многословный                  | компактный                          |
| Тип                | создаёт новый класс           | не создаёт — просто ссылка на метод |
| `this`             | относится к анонимному классу | относится к внешнему объекту        |
| Производительность | чуть хуже                     | эффективнее (инлайнится JVM)        |

Лямбды — это "syntactic sugar" над MethodHandle, что делает их более эффективными, чем анонимные классы.


---

## Пример

    interface Operationable {
        int calculate(int x, int y);
    }
    
    public static void main(String[] args) {
        Operationable operation = (x, y) -> x + y;     
        int result = operation.calculate(10, 20);
        System.out.println(result); //30
    }

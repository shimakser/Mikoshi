# Dockerfile networks

Docker-сети обеспечивают взаимодействие контейнеров между собой и с внешним миром.

## Основные типы сетевых драйверов
- bridge — сеть по умолчанию.
  - Docker создает виртуальную локальную сеть, где все контейнеры могут общаться друг с другом напрямую.
  - Контейнеры в этой сети видят друг друга по внутренним IP-адресам, но снаружи (из твоей основной сети или интернета) они недоступны, если ты специально не откроешь порты.
- host — контейнер использует сетевой стек хоста напрямую, без виртуальной сети. Это значит, что контейнер и твой компьютер делят один и тот же IP-адрес и порты. Если контейнер запускает веб-сервер на порту 80, то этот порт будет доступен сразу на твоем компьютере без дополнительного проброса.
- none — отключает все сетевые интерфейсы контейнера, изолируя его полностью.
- overlay — позволяет объединить контейнеры, запущенные на разных Docker-хостах, в одну виртуальную сеть. Используется в Docker Swarm и других оркестраторах.
- macvlan — контейнеру присваивается MAC-адрес, и он выглядит как отдельное физическое устройство в сети, что позволяет интегрировать контейнеры в существующую сеть на уровне L2.

## Особенности и рекомендации
- Bridge-сеть подходит для изолированных приложений на одном хосте, обеспечивая простое взаимодействие контейнеров.
- Overlay-сети используются для мульти-хостовых сценариев, например, в кластерах Docker Swarm.
- Host-сеть убирает сетевую изоляцию, что может быть полезно, но снижает безопасность.

# Reverse Proxy Server

Это сервер, который выступает в роли посредника между клиентами (пользователями) и внутренними веб-серверами, принимая входящие запросы от клиентов и перенаправляя их на соответствующие backend-серверы. В отличие от прямого прокси, который работает на стороне клиента, обратный прокси располагается на стороне сервера и скрывает архитектуру внутренних серверов от внешних пользователей.

## Принцип работы
- Получение запроса: Принимает HTTP/HTTPS запросы от клиентов
- Анализ запроса: Определяет, какой backend-сервер должен обработать запрос
- Перенаправление: Отправляет запрос на соответствующий внутренний сервер
- Получение ответа: Получает ответ от backend-сервера
- Возврат клиенту: Передает ответ обратно клиенту
  
Для клиента это выглядит так, будто он напрямую взаимодействует с веб-сервером, не подозревая о существовании промежуточного слоя.

---

## Основные функции и возможности
1. Балансировка нагрузки.
2. Кэширование — Обратный прокси может кэшировать статический и динамический контент, что значительно уменьшает нагрузку на основные серверы и ускоряет время отклика.
3. SSL Offloading — Прокси может взять на себя обработку SSL/TLS-шифрования, освобождая backend-серверы от ресурсоемких операций шифрования/дешифрования.
4. Безопасность
   - Скрытие реальных IP-адресов внутренних серверов.
   - Защита от DDoS-атак.
   - Фильтрация вредоносного трафика.
   - Применение правил безопасности на уровне приложений.

---

## Преимущества
1. Масштабируемость — Позволяет легко добавлять новые backend-серверы без изменения клиентского кода.
2. Отказоустойчивость — При выходе из строя одного сервера трафик автоматически перенаправляется на работающие серверы.
3. Централизованное управление — Единая точка входа для всех клиентских запросов упрощает мониторинг и управление.
4. Упрощение SSL-управления.

## Недостатки и ограничения
1. Единая точка отказа.
2. Дополнительная задержка.
3. Сложность отладки.

## Отличия от Load Balancing
1. **RP**
   - Работает на 7 уровне OSI (уровень приложений), исключительно с HTTP/HTTPS трафиком.
   - Редоставляет полезные функции даже при работе с одним сервером.
   - Фокусируется на оптимизации, безопасности и кэшировании.
2. **LB**
   - Может работать на разных уровнях OSI — от 3-го до 7-го уровня.
   - Специализируется на равномерном распределении входящего трафика между несколькими серверами — требует наличия нескольких серверов для эффективной работы.
   - Основная цель — высокая доступность и отказоустойчивость.

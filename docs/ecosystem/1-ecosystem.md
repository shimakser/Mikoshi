# Java ecosystem

---

## JDK vs JRE vs JVM
- **JDK** — комплект для разработки: включает JRE + компилятор и инструменты (например javac, javadoc).
- **JRE** — окружение для запуска Java-программ: включает JVM + стандартные библиотеки.
- **JVM** — виртуальная машина, исполняет байт-код (.class).
- **javac** — javac — компилятор, переводит исходники (.java) в байт-код (.class), который потом выполняет JVM.

---

## Процесс создания Java-программы

```mermaid
    A[Исходный код .java] --> B[Java Compiler javac]
    B --> C[Байт-код .class]
    C --> D[Java Virtual Machine JVM]
    D --> E[Машинный код]
    E --> F[Выполнение программы]
    
    G[IDE] --> A
    H[Build Tool Maven/Gradle] --> B
    I[Package Manager] --> C
```

## Жизненный цикл Java-программы

1. **Компиляция** — **Компилятор** (`javac`) проверяет синтаксис и и компилирует (.java → .class).
2. **Загрузка** — **ClassLoader** загружает байт-код в память JVM.
3. **Верификация** — JVM проверяет корректность и безопасность загруженного кода.
4. **Выполнение** — JIT-компилятор (часть JVM) преобразует часто используемые участки байт-кода в машинный код для ускорения.
5. **Сборка мусора** — JVM автоматически управляет памятью, освобождая неиспользуемые объекты.
